SVM algorithm parameters

(float) Penalty parameter C of the error term.
(string) Specifies the kernel type to be used in the algorithm.
(int) Degree of the polynomial kernel function ('poly'). Ignored by all other kernels.
(float) Kernel coefficient for 'rbf', 'poly' and 'sigmoid'. If gamma is 'auto' then 1/n_features will be used instead.
(float) Independent term in kernel function. It is only significant in 'poly' and 'sigmoid'.
Enable probability estimates. (boolean) This must be enabled prior to calling fit, and will slow down that method.
Use the shrinking heuristic. (boolean)
(float) Tolerance for stopping criterion.
(float) Specify the size of the kernel cache (in MB).
(dict, 'balanced') Set the parameter C of class i to class_weight[i]*C for SVC. If not given, all classes are supposed to have weight one. The 'balanced' mode uses the values of y to automatically adjust weights inversely proportional to class frequencies in the input data as n_samples / (n_classes * np.bincount(y)).
Enable verbose output. (boolean) Note that this setting takes advantage of a per-process runtime setting in libsvm that, if enabled, may not work properly in a multithreaded context.
(int) Hard limit on iterations within solver, or -1 for no limit.
(string) Whether to return a one-vs-rest ('ovr') decision function of shape (n_samples, n_classes) as all other classifiers, or the original one-vs-one ('ovo') decision function of libsvm which has shape (n_samples, n_classes * (n_classes - 1) / 2). The default of None will currently behave as 'ovo' for backward compatibility and raise a deprecation warning, but will change 'ovr' in 0.18.
New in version 0.17: decision_function_shape='ovr' is recommended.
Changed in version 0.17: Deprecated decision_function_shape='ovo' and None.
(int or None) The seed of the pseudo random number generator to use when shuffling the data for probability estimation.
